# Contributing to File Converter Pro

Thank you for your interest in contributing! We welcome all kinds of contributions.

## 🎯 Ways to Contribute

- 🐛 **Report Bugs** - Found an issue? Let us know!
- 💡 **Suggest Features** - Have an idea? Share it!
- 📝 **Improve Documentation** - Help us document better
- 🔧 **Submit Code** - Fix bugs or add features
- 🧪 **Test** - Help test new features

## 🚀 Getting Started

### 1. Fork the Repository
Click the "Fork" button on the GitHub repository page.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR-USERNAME/File-Converter.git
cd File-Converter
```

### 3. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
# or for bug fixes:
git checkout -b bugfix/issue-description
```

### 4. Set Up Development Environment
```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 5. Make Your Changes
Edit the code and test thoroughly.

### 6. Commit Your Changes
```bash
git add .
git commit -m "Add: Brief description of your changes"
```

### 7. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 8. Create a Pull Request
Go to the original repository and create a PR with a clear description.

## 📋 Contribution Guidelines

### Code Style
- Follow PEP 8 Python style guide
- Use meaningful variable names
- Add comments for complex logic
- Keep functions small and focused

### Commit Messages
- Use clear, descriptive messages
- Start with action: "Add", "Fix", "Update", "Remove"
- Example: `Add: PDF to JPG conversion feature`

### Pull Request Requirements
- Clear title and description
- Reference related issues (#123)
- Include screenshots for UI changes
- Test your changes before submitting

## 📝 Reporting Bugs

Include the following in bug reports:
- **Description**: What is the bug?
- **Steps to Reproduce**: How to recreate the issue
- **Expected Behavior**: What should happen
- **Actual Behavior**: What actually happens
- **Environment**: OS, Python version, browser
- **Screenshots**: If applicable

## 💡 Suggesting Features

Include:
- **Description**: What would you like added?
- **Use Case**: Why is this useful?
- **Implementation Ideas**: Any suggestions?
- **Examples**: How would users interact with it?

## 🤝 Community

- Be respectful and inclusive
- Assume good intentions
- Focus on the code, not the person
- Help others learn and grow

## ❓ Questions?

Open an issue with the label `question` or start a discussion!

## ✅ Review Process

1. We review all PRs within 3-5 days
2. We may ask for changes or improvements
3. Once approved, your PR will be merged
4. You'll be credited in the changelog!

## 🎉 Thank You!

Your contributions make this project better for everyone. We appreciate your help! 🙌
